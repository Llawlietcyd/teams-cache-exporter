# 商店上架准备清单

这份文档用于整理把扩展发布到浏览器商店前需要准备的事项。

## 发布前先确认

- 确认仓库名和扩展名已经最终定稿
- 替换占位版权信息和联系方式
- 确认扩展名称足够明确，不容易与现有产品混淆
- 确认图标已经是最终版本
- 重新检查所有请求的权限
- 确认仓库里没有个人聊天导出文件

## 浏览器商店需要准备的内容

### Chrome Web Store

需要准备：

- 最终版 `manifest.json`
- 扩展图标
- 截图
- 短描述
- 长描述
- 隐私政策 URL
- 支持页面 URL 或仓库 URL

### Microsoft Edge Add-ons

需要准备：

- 同一份扩展包
- 隐私政策 URL
- 截图
- 上架文案

## 推荐文案

建议的短介绍：

> Export the currently open Microsoft Teams web chat from browser cache into cleaned JSON, CSV, and Markdown.

如果要写中文版本，可以用：

> 从浏览器缓存导出当前打开的 Microsoft Teams 网页聊天，并生成清洗后的 JSON、CSV 和 Markdown 文件。

## 可能被审核追问的点

- 访问 Teams 内容可能被视为敏感的企业或学校数据
- 审核人员可能会问为什么需要 Teams 域名的 host 权限
- 审核人员可能会问数据是否离开浏览器

你需要准备好明确说明：

- 扩展只导出当前打开的聊天
- 所有处理都在本地完成
- 没有第三方后端
- 下载文件直接保存在用户自己的设备上

## 上架前还建议补的内容

- 补一套更完整的截图
- 补商店素材图
- 增加 support / issues URL
- 增加 changelog
- 同时在 Chrome 和 Edge 上做测试
- 尽量测试不同 Teams 租户
- 再检查一遍弹窗文案是否和最终导出行为一致

## 还可以继续打磨的方向

- 虽然标准扩展 API 不能自定义下载目录，但可以进一步优化提示文案
- 如果缓存中有更多 profile 数据，可以继续增强参与者姓名解析
- 可以补更细的进度状态，比如“读取缓存”“清洗内容”“下载文件”
